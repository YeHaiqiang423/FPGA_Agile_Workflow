# 孩子们，我来教你们怎么用vscode + Github + vivado（Windows版）+ iverilog + surfer（看波形）+ Tcl/shell 进行联动开发

## 说在前面

本教程旨在抛弃vivado界面，让UI归UI，引擎归引擎，提升开发效率。

我们会使用到的技能、软件、插件如下：
1. VS Code (宇宙第一编辑器) + WSL2 

考虑到大家大多数为Windows操作系统开发，为了防止下载配置虚拟机带来的挫败感，我们直接使用Windows 里的 WSL2 (Ubuntu 系统)（WSL全称为Windows Subsystem of Linux，是不是很好理解喵）

2. Vivado (Windows版) / iverilog

我们再也不会去点开它的图形界面（除非去配IP核），而是通过命令行悄悄唤醒它的底层仿真引擎 (XSIM) 去帮我们跑数据。在这里要说清楚的是，iverilog本身是开源工具，是无法解析vivado的加密IP核的，所以我们还是要使用XSIM进行仿真，iverilog本身是一个辅助你检查语法的编译器。

3. Git & GitHub

解决多人协作冲突，记录每一次代码快照。代码写炸了一键回退，永远告别压缩包乱飞的时代。队友很舒服。

4. TerosHDL 插件（很牛逼的插件，我也没能太深入研究，但是目前我会的东西足以大大提升效率）

它能在 VS Code 里自动帮你补全 Verilog 代码、画出模块层级树，甚至不需要经过 Vivado 综合，就能直接给你渲染出 RTL 门级电路连线图。

5. Surfer

仿真跑完瞬间，直接在 VS Code 里点开波形文件（支持极限压缩的 .fst 格式），缩放平移毫无卡顿，再也不用忍受 Vivado 自带波形查看器的龟速。此外，也可以通过Tcl脚本直接渲染真实开发板上 ILA 抓取并导出的波形数据。
***

稍微上帝视角说明一下流程：

1. 获取代码与编写（Git + VS Code + TerosHDL）

每天开工第一件事，在终端敲 git pull 拉取队友的最新进度。

在 VS Code 里愉快地敲 Verilog 代码。TerosHDL显示Hierarchy，模块关系一目了然。

2. 一键无头仿真（WSL2 + Bash/Tcl + Vivado XSIM）

代码写完，直接在 VS Code 底部的终端（WSL 环境）里敲一行脚本运行命令，比如 ./quicksim.sh。
Windows 里的 Vivado 底层引擎 (XSIM)启动，跑完Testbench后退出，并在文件夹里留下波形文件（.fst 或 .vcd）。

3. 秒开波形分析（Surfer）

在 VS Code 的文件树里找到刚刚生成的波形文件，鼠标点击。

Surfer 插件瞬间接管，波形直接在代码旁边的分屏里渲染出来。哪里不对改哪里，改完再按一下上方向键重新运行脚本，实现“修改 -> 运行 -> 看波形”的极速闭环。

4. 存档并同步（Git & GitHub）

模块跑通了，波形完美了。

在终端敲下 git add . 和 git commit 将代码放入本地保险箱。

敲下 git push 发射到云端。队友立刻就能用你的最新成果。

## Git & Github 篇

### 一、看懂Git核心原理

将Git视作有4个完全隔离的区域
1. **工作区** 就是你现正在vscode里面编辑的区域
2. **暂存区** 相当于打包托盘，你改了 10 个文件，但只觉得其中 3 个写得完美了，就把这 3 个挑出来放进托盘，剩下的继续在工作区留着修改。
3. **本地仓库** ~~旮旯给木本地存档（学习版）~~ 把托盘里的文件永远定格成一个不可篡改的历史记录
4. **远程仓库** ~~旮旯给木云存档（Steam版）~~ 也就是 GitHub / Gitee。把你的本地存档上传到云端服务器，队友就能看到了。

### 二、首次配置
刚装好 Git，如果不告诉 Git 你是谁，它会为了防止匿名破坏而死活不让你存档。

打开 VS Code 的终端，输入下面两行指令：
(⚠️ 注意：带 【 】 的地方，请连同方括号一起替换为你自己的真实信息！)

```bash
# 告诉 Git 你的代号/名字
git config --global user.name "【你的拼音或英文名】" 

# 告诉 Git 你的邮箱（随便填个格式正确的就行，比如 qq 邮箱）
git config --global user.email "【你的邮箱@qq.com】"
```
### 三、 场景实战字典
不要去背 Git 复杂的指令，你只需要知道你想干什么，然后来这里查对应的命令。
***
**场景 A：我是项目发起人，我要把本地的新工程推上云端**

1. 在项目根目录初始化 Git（赋予这个文件夹 Git 的灵魂）：

```bash
git init
```
2. 把当前目录所有文件放进“托盘”：

```bash
git add .
```

3. 拍下第一张快照（存档）：

```bash
git commit -m "【填入你想说的话，比如：初版框架搭建】"
```
4. 给云端大厅绑定一个地址：

```bash
# 把后面那串网址替换成你在 GitHub 上新建的空仓库的 HTTPS 链接
git remote add origin 【https://github.com/你的名字/你的仓库名.git】
```

5. 暴力推送到云端主分支：

```bash
git push -u origin main
```
***
**场景 B：我是队友，我要加入开发**

你不需要搞场景 A 那一套，你只需要找个空文件夹，把云端代码“吸”下来。

```bash
# 把网址替换成你们团队实际的仓库链接
git clone 【https://github.com/你的名字Manba/你的仓库名Out.git】
```

执行完后，VS Code 里直接 File -> Open Folder 打开克隆下来的文件夹，直接开干！

当然有时候会遇到这个问题：ABC三个人都在原有的分支上开发，如果只保留一套main分支会给项目维护者带来很大的麻烦而且如果缺乏沟通会导致项目进度拖延
解决方案是，ABC三个人各自开一套与main并行的分支进行开发，随后同步进度，三个人提交PR，由一个聪明的审核人审核冲突，并且告知三个人怎么解决，解决冲突后三个人的代码就可以合并到main分支里了
clone下一个项目后，签出一个分支

```bash
git checkout -b (branch name)
```
当你在这个分支上进行一些开发后，也可以随时切回到main分支
```bash
git checkout main
```
这时候你会发现你做的一些修改消失了，因为此时是main分支

为了保持项目进度同步，因为别人也在开发，需要同步下main分支的更新

```bash
git fetch origin
git merge origin/main
```

可能会有一些冲突因为别人改了和你一样的地方，这时候需要和别人商量一下

你的分支完成后，可以正积极性之前的几步，提交到你的分支上

这时候给项目维护者提一个PR，就可以合并到main了，顺便你也可以成为这个项目的`contributor`

但是这样做的前提是需要你是这个项目的合作者，也就是需要项目维护者交给你权限你才能对这个仓库写入

那么如何维护一个开源项目呢，那就是把项目先`fork`下来到自己的仓库中，然后进行开发，此时是main也可以因为
它是fork的，不影响原仓库。你可以随便提交因为这是你的仓库。

随后给项目维护者发PR就可以，同样也可以成为`contributor`

有人给pytorch找错别字都成为了contributor，然后在简历上大写一笔

大家给这个仓库发给PR吧！

***
**场景 C：日常开发的 4 步连招（每天必须严格执行喵）**

写代码的每一天，都必须死记硬背这四个动作的顺序，绝对不能漏掉第一步！

1. 拉取最新代码（开工第一件事）

坐到电脑前，必须先把队友昨晚的修改同步下来，否则你们同时改同一行代码会触发极其痛苦的冲突！

```bash
git pull
```
2. 查看状态（随时可用）

写了一半，看看自己到底改了啥，哪些红了（没进托盘），哪些绿了（在托盘里）。

```bash
git status
```
3. 放入托盘并存档（下班前干的事）

把今天所有改动装盘，并写上人能看懂的留言。

```bash
git add .
git commit -m "【描述你干了啥，比如：修复了 FIR 滤波器的位宽溢出 bug】"
```

4. 推送到云端（完事收工）

发射到 GitHub，让队友能看见。

```bash
git push
```

场景 D：我把代码改烂了，需要后悔药(~~旮旯给木里不是这样的，我应该直接国一的。。。~~)


### 硬件团队的 Git 铁律（极其重要）
在我们的 FPGA/Vivado 联合开发中，谁要是敢执行 git add . 把临时文件传上去，直接肘击他好吧

Vivado 每次编译会产生几个 GB 的 *.log、.jou、.sim 等垃圾缓存，波形文件 .vcd 也是几百 MB。绝对不准把这些传上云端喵

解决方案：
在项目**最外层目录**建一个名为 .gitignore 的文件，注意哦最外层。把你不想传的后缀名写进去，Git 就会假装看不见它们。我们团队的标准模板如下：

```plaintext
# Vivado 缓存与生成产物
*.jou
*.log
*.pb
*.str
.Xil/
*.cache/
*.sim/
*.hw/
*.ip_user_files/

# 仿真波形垃圾
*.vcd
*.fst
*.vvp
*.out

# 永雏塔菲
```
